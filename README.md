## Demo Scenario

A pharmaceutical company is collecting data from clinical research: adverse events, patients, and trial outcomes. We use Genie Code to build the full pipeline.

**All notebooks in `ai-generated-demos/` were generated by Genie Code**, not hand-written. The only manual step was the synthetic data generation (`setup/0_pharma_data_generation.py`). Everything else was produced through conversations with the AI agent, each demonstrating a different prompting pattern:

| Step | Agent / Feature | Notebook | Prompting Pattern |
|------|----------------|----------|-------------------|
| **Data Generation** | _(manual)_ | `setup/0_pharma_data_generation.py` | Pre-requisite |
| **Data Engineering** | Data Engineering Agent | `ai-generated-demos/1_data_engineering.py` | Instructions + No Plan |
| **Dashboard** | Dashboard Agent _(Preview)_ | _(created live)_ | Plan + No Instructions |
| **Data Science** | Data Science Agent | `ai-generated-demos/2_...` + `ai-generated-demos/3_...` | Instructions + Plan + Interrupt |
| **Genie Space** | Custom Skill | `ai-generated-demos/4_genie_space_custom_skill.py` | Skill-triggered |

### Data Model

Synthetic clinical trial data with baked-in predictive signal for the ML classification task.

- **`patients.json`** (2,000 records): demographics, BMI, pre-existing conditions, baseline labs
- **`clinical_trials.json`** (15 records): trial metadata, drug info, phases, sponsors
- **`trial_results.json`** (8,000 records): per-visit treatment outcomes, vitals, lab values
- **`adverse_events.json`** (5,000 records): adverse events with severity, risk scores, and `serious` flag (ML target)

### Medallion Architecture

Built by the Data Engineering Agent using Spark Declarative Pipelines (SDP):

- **Bronze**: Streaming tables via Auto Loader (`cloudFiles`)
- **Silver**: Materialized views with type casting, data quality expectations, CDF
- **Gold**: BI-ready aggregates: trial safety summary, patient outcomes, drug efficacy

### ML Target

- **Task**: Binary classification to predict `serious` adverse events
- **Model**: LightGBM with Hyperopt tuning, bundled into a custom PyFunc
- **Primary metric**: ROC-AUC
- **Key insight**: `risk_score` causes data leakage (derived from target), detected and removed during the demo

## Customization Layers Demonstrated

| Layer | What It Does | Example in This Demo |
|-------|-------------|---------------------|
| **Instructions** | Always-on rules for every conversation | [`setup/assistant_instructions.md`](setup/assistant_instructions.md): SDP conventions, MLflow tracking, UC namespace |
| **Skills** | On-demand expertise loaded when relevant | `prompt-to-genie` skill for Genie Space creation |
| **MCP** | External tool access via UC-governed connections | GitHub MCP for repository operations |

## Prerequisites

- Databricks workspace with Unity Catalog enabled
- Serverless compute or a cluster with `category_encoders`, `lightgbm`, `hyperopt`
- Permissions to create schemas, volumes, and models in your target catalog
- Genie Code enabled in the workspace

## Quick Start

1. Clone this repo and push notebooks to your Databricks workspace
2. Run `setup/0_pharma_data_generation.py` to generate synthetic data
3. Follow the [RUNBOOK.md](RUNBOOK.md) for the full walkthrough
